# 뉴스 대시보드 실행 계획

## 팀
- 사회혁신가: 이동이
- 개발자: 정장영

## 문제
- 실제 업무 문제: 애드보커시 단체는 사업 성과지표로 언론보도 횟수를 정하는 경우가 많지만, 현재 단체 관련 키워드를 구글 알리미로 받아 업무 메신저 잔디에 흘려보내는 방식이라 보도 내역이 아카이빙되지 못하고 휘발된다. 평가 시기에는 실제 언론보도 횟수와 근거 기사를 측정하기 어렵다.
- 가장 어려움을 겪는 사람: 사업 담당자인 활동가, 평가해야 하는 동료 및 사무처장
- 문제가 발생하는 순간: 사업 평가 시 해당 사업의 언론 영향력과 사회적 영향력을 설명해야 하는데, 보도 내역이 아카이빙되어 있지 않아 근거가 부족해지는 순간

## 자료 확인
- 자료 상태: 공개 언론 기사 기반
- 샘플 입력: 활동명 '나무의 권리 재인식 플랫폼: 시티트리클럽'과 핵심 키워드 '서울환경연합, 서울환경운동연합, 가로수, 시티트리클럽, 나무의 권리'
- 개인정보 처리: 단체 내부자료는 사용하지 않고 공개 언론 기사와 공개 키워드만 사용한다.
- 자료 유효성 확인: 3시간 안에는 사용자가 수기로 찾은 Google Sheets 목록 중 2026년 1월 1일 이후 `분류=기사`인 공개 언론사 기사 7건을 최소 입력 묶음으로 사용한다. 서울환경연합 홈페이지, 서울시공익활동지원센터처럼 언론사가 아닌 출처와 삭제 링크는 제외한다. 내부 문서 원문, 비공개 평가 자료, 개인정보는 입력하지 않는다.

## 현재 업무 흐름
- 입력 자료: 구글 알리미로 들어오는 단체 관련 키워드 뉴스 알림, 잔디 업무 메신저에 남는 알림 메시지, 공개 언론 기사
- 처리 순서: 구글 알리미 수신 -> 잔디 메시지 확인 -> 필요한 경우 사람이 기억하거나 별도 저장 -> 평가 시기에 보도 내역을 다시 찾음
- 판단 기준: 기사가 해당 사업 활동과 실제로 관련 있는지, 보도 횟수에 포함해도 되는지, 평가 설명에 쓸 대표 기사인지 사람이 판단한다.
- 현재 산출물: 현재는 안정적인 아카이브나 성과평가용 보도 목록이 없다.
- 현재 사용 도구: 구글 알리미, 잔디, 수동 검색

## 결과물 형태
- 문제 유형: 데이터 정리/분석/대시보드
- 선택한 결과물 형태: 대시보드 + 성과평가용 요약 리포트
- 선택 이유: 사업 평가 시점에 보도 횟수, 대표 기사, 관련성 판단, 평가용 설명 문장을 한 번에 확인해야 하므로 화면형 대시보드와 요약 리포트를 함께 보여주는 방식이 가장 자연스럽다.

## 기대 변화
- 기대 변화: 공개 기사 검색 결과를 사업 활동 기준으로 정리해 평가 시점에 언론보도 횟수와 대표 근거를 바로 확인한다.
- 관찰 가능한 개선: 시티트리클럽 관련 공개 기사 목록, 보도 횟수, 대표 기사, 관련성 표시, 평가용 요약 문장이 한 화면과 리포트 문장으로 나온다.

## 3시간 MVP
- 해결할 좁은 문제: 시티트리클럽 활동과 핵심 키워드에 대한 공개 뉴스 보도 목록을 수집 또는 입력해, 관련 기사 여부와 보도 횟수 및 평가용 설명 문장을 검토 가능한 형태로 만든다.
- 핵심 기능 1개: 활동명과 키워드에 맞는 공개 기사 묶음을 평가자가 검토할 수 있는 뉴스 성과 대시보드 한 화면으로 정리한다.
- 포함할 업무 단계: 활동명/키워드 입력 -> 공개 기사 목록 정리 -> 키워드 및 관련성 매칭 -> 보도 횟수와 대표 기사 표시 -> 평가용 요약 문장 생성 -> 담당자 검토
- 데모 입력: 활동명 '나무의 권리 재인식 플랫폼: 시티트리클럽'과 키워드 '서울환경연합, 서울환경운동연합, 가로수, 시티트리클럽, 나무의 권리', 그리고 수기 시트에서 확인한 2026년 1월 1일 이후 공개 언론사 기사 URL 7건
- 데모 산출물: 뉴스 아카이빙 대시보드와 성과평가용 요약 리포트 초안
- AI 역할: 공개 기사 목록 정리, 키워드 매칭, 중복 의심 표시, 관련성 판단 보조, 평가용 설명 문장 초안 생성
- 사람 검토 지점: 사업 담당자와 평가자가 관련 기사 여부를 확인하고, AI가 만든 언론 영향력 설명 문장이 과장 없이 맞는지 검토한다.
- 성공 기준: 시티트리클럽과 핵심 키워드를 넣었을 때 대시보드에 공개 기사 목록, 보도 횟수, 대표 기사, 관련성 표시, 평가용 요약 문장이 나오면 성공이다.
- 실패 신호: 활동과 무관한 기사를 성과로 세거나, 비언론사 출처를 언론보도로 집계하거나, 같은 언론사 안의 동일/반복 기사를 중복 집계하거나, 평가 문장이 실제 기사보다 과장되면 실패로 본다.
- 3시간 안/밖 판단: 안 - 공개 기사와 키워드를 기준으로 수동 입력 또는 공개 검색 결과 묶음만 다루고, 자동 크롤링/로그인/DB 저장/잔디 연동은 제외하면 3시간 안에 검토 가능한 데모가 가능하다.

## 제외 범위
- 오늘 하지 않을 것: 구글 알리미 자동 연동, 잔디 메시지 자동 수집, 실시간 크롤링, 로그인/권한, 운영 DB 저장, 전체 사업별 장기 아카이브, 최종 성과 판단 자동화
- 다음 단계 아이디어: 구글 알리미 메일/잔디 메시지에서 기사 링크를 자동 추출하기, 월별/사업별 아카이브 저장, 중복 기사 병합, 평가보고서 문단 자동 생성

## 3시간 구현 계획
- 0-30분: 시티트리클럽 키워드와 데모 기사 입력 형식을 고정하고, 대시보드에 보여줄 필드(제목, 언론사, 날짜, URL, 매칭 키워드, 관련성, 대표 여부)를 정한다.
- 30-90분: 공개 기사 URL 또는 검색 결과 묶음을 입력받아 기사 목록 테이블로 정리하고, 키워드 매칭과 중복 의심 표시를 만든다.
- 90-150분: 보도 횟수, 대표 기사, 관련성 검토 상태, 평가용 요약 문장을 보여주는 대시보드 화면 또는 정적 웹페이지를 만든다.
- 150-180분: 샘플 기사 묶음으로 성공/실패 기준을 검증하고, 담당자가 관련성 및 평가 문장을 검토할 수 있도록 표시를 다듬는다.

## 구현자 지침
- 구현 시작 전 이 `PLAN.md`와 `workshop.json` 초안을 먼저 읽습니다.
- 새 세션에서 구현하더라도 구현 중 결정, 변경, 막힌 점, 검증 결과를 `MEMORY.md`에 append-only로 기록합니다.
- `MEMORY.md`가 없으면 새로 만들고, 있으면 기존 내용을 덮어쓰지 않고 파일 끝에만 추가합니다.
- 구현이 끝나면 `workshop.json`의 구현 결과 관련 필드를 갱신할 수 있게 변경 요약과 검증 결과를 남깁니다.

## 빠르게 끝났을 때 이어서 할 작업 후보
- 후보 1 작업: 중복 기사 병합과 보도 횟수 산정 기준 강화
- 후보 1 시작 조건: 기본 대시보드가 공개 기사 묶음에서 정상 표시되고 40분 이상 남았을 때
- 후보 1 구현 내용: 같은 제목/URL/언론사 조합을 기준으로 중복 의심 기사를 표시하고, 집계에 포함/제외할 수 있는 상태값을 추가한다.
- 후보 1 데모 산출물: 중복 의심 표시와 보도 횟수 집계 기준이 포함된 기사 목록
- 후보 1 예상 추가 시간: 40분
- 후보 1 검수 기준: 동일 기사 또는 유사 기사로 보이는 항목을 담당자가 확인하고 집계 포함 여부를 판단할 수 있다.
- 후보 2 작업: 성과평가용 리포트 문단 내보내기
- 후보 2 시작 조건: 대표 기사와 보도 횟수 집계가 안정되고 30분 이상 남았을 때
- 후보 2 구현 내용: 대시보드 요약을 바탕으로 평가보고서에 붙여 넣을 수 있는 3-5문장 리포트 초안과 근거 기사 목록을 생성한다.
- 후보 2 데모 산출물: 평가보고서용 언론 영향력 요약 문단과 근거 기사 목록
- 후보 2 예상 추가 시간: 30분
- 후보 2 검수 기준: 요약 문장이 보도 횟수와 대표 기사 근거를 반영하며, 과장 표현 없이 사람이 바로 수정할 수 있다.
