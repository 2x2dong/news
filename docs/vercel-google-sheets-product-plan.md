# 뉴스 대시보드 운영 기획서

## 한 줄 요약

서울환경연합의 연간 사업계획서를 기준으로, 조직과 캠페인이 언론·블로그·SNS·기관 게시물에서 어떻게 언급되는지 모으고, 사람이 검토해 성과 평가에 쓸 수 있게 정리하는 저비용 뉴스 아카이브입니다.

## 왜 필요한가

현재는 구글 알리미와 잔디로 단체 관련 뉴스가 흘러오지만, 시간이 지나면 아카이빙되지 않아 평가 시점에 보도 횟수와 대표 근거를 다시 찾기 어렵습니다.

특히 사업계획서에는 여러 사업과 캠페인이 있고, 일부 사업은 언론보도 횟수뿐 아니라 콘텐츠 재생산, 사회적 확산, 시민 반응도 함께 봐야 합니다. 따라서 기사만 세는 도구가 아니라, 여러 출처를 저장하되 성과 집계 기준을 분리하는 운영 도구가 필요합니다.

## 기본 방향

우선순위는 `기사 중심 아카이브`입니다.

1. 모든 수집 결과를 먼저 저장합니다.
2. 각 결과에 출처 유형, 관련 조직, 관련 캠페인, 관련 사업 후보를 붙입니다.
3. 사람이 검토해 집계 포함 여부와 대표 근거 여부를 확정합니다.
4. 사업별 대시보드는 저장된 기사/게시물 위에서 분석 화면으로 제공합니다.

사업별 화면만 먼저 만드는 방식은 후순위입니다. 데이터가 먼저 잘 모여야 사업별 성과 화면도 신뢰할 수 있습니다.

## 운영 비용 기준

고정 비용을 최소화합니다.

- 화면: Vercel 무료 범위에서 시작
- 저장소: Google Sheets를 DB처럼 사용
- 자동 수집: Google Apps Script 시간 트리거
- 수집 주기: 하루 2회, 오전 1회와 저녁 1회
- 수동 수집: 관리자 화면에서 버튼으로 재수집 가능
- 향후 이전: Sheets가 느려지거나 데이터가 너무 많아지면 그때 Postgres/Supabase 같은 DB로 이전

처음부터 별도 DB를 붙이지 않습니다. 소규모 조직에서는 Google Sheets가 비개발자 관리와 비용 측면에서 더 적합합니다.

## 사용자와 권한

권한은 두 단계만 둡니다.

### admin

- 관리자 페이지 접근
- 사업계획서 업로드
- 검색어 추가, 수정, 비활성화
- 수동 기사 수집 실행
- 기사/게시물 검토 상태 변경
- 집계 포함 여부 변경
- 대표 기사 지정
- AI 매칭 결과 확정 또는 수정

### viewer

- 대시보드 조회
- 기사/게시물 목록 조회
- 사업별 요약 조회
- 수정 불가

사업 담당자별 권한은 만들지 않습니다. 조직 규모상 `admin/viewer` 두 단계가 충분합니다.

## 사업계획서 관리

사업계획서는 매년 2월 전후 작성되고 이후 거의 업데이트되지 않습니다.

- 매년 관리자가 새 Google Docs 링크 또는 export 텍스트를 업로드합니다.
- 자동 동기화는 하지 않습니다.
- 업로드한 내용은 해당 연도의 사업계획서 스냅샷으로 저장합니다.
- 연도별로 사업 목록, 활동명, 변화목표, 관찰지표, 산출지표, 담당자, 협력기관을 추출합니다.

## 수집 범위

모두 저장하되 유형을 분리합니다.

### 저장하는 것

- 언론사 기사
- 블로그
- SNS 게시물
- 기관 게시물
- 단체 자체 게시물

### 집계 기준

- `보도 횟수`에는 언론사 기사만 포함합니다.
- 블로그, SNS, 기관 게시물은 사회적 확산이나 콘텐츠 재생산 근거로 별도 활용합니다.
- 단체 자체 게시물은 보도 성과가 아니라 활동 기준/근거 자료로 분리합니다.

## 기사 저장 범위

원문 전체는 저장하지 않습니다.

저장하는 정보:

- 제목
- URL
- 언론사 또는 출처명
- 게시일
- 검색어
- 출처 유형
- 스니펫
- AI 요약
- 매칭 근거
- 사람이 검토한 상태

필요하면 원문에서 짧은 근거 문장 일부만 저장합니다. 평가나 최종 확인은 원문 링크를 열어 사람이 확인합니다.

## 검색어 생성 기준

사업계획서 전체 문장을 키워드로 쓰지 않습니다. 그러면 너무 많은 기사가 딸려옵니다.

### 항상 검색하는 조직명

- 서울환경연합
- 서울환경운동연합

### 자동 추출할 후보

사업계획서에서 고유명사 캠페인/브랜드명을 후보로 추출합니다.

예:

- 플라스틱방앗간
- 시티트리클럽
- 씨앗의숲

### 관리자가 할 수 있는 일

- 검색어 추가
- 검색어 수정
- 검색어 비활성화
- 검색어별 유형 지정: 조직명, 캠페인명, 사업명, 협력기관명 등

## 환경연합 유사 검색어 처리

`환경운동연합`, `환경연합`은 전국 조직과 지역 조직이 많아 잡음이 큽니다.

예:

- 파주환경운동연합
- 부산환경연합
- 지역 환경운동연합 지부

따라서 기본 원칙은 다음과 같습니다.

- `서울환경연합`, `서울환경운동연합`은 저장 후보로 봅니다.
- 지역 지부명만 있는 기사는 서울환경연합 성과로 자동 집계하지 않습니다.
- 다만 `시티트리클럽`, `플라스틱방앗간`, `씨앗의숲` 같은 관리 중인 캠페인 고유명사가 함께 나오면 저장합니다.
- 이 경우 `캠페인명 매칭`, `지역/전국 조직 혼재 가능성`으로 표시하고 사람이 검토합니다.

## AI 매칭 기준

자동 확정이 아니라 AI 추천 + 사람 확정입니다.

1. 규칙 기반 1차 후보 매칭
   - 조직명
   - 캠페인명
   - 사업명
   - 활동명
   - 협력기관명

2. AI 2차 판정
   - 어떤 사업과 관련 있는지
   - 직접 관련인지 간접 관련인지
   - 조직 전체 언급인지
   - 왜 그렇게 판단했는지

3. 사람 최종 확인
   - 집계 포함 여부
   - 대표 기사 여부
   - 중복 여부
   - 평가 문장에 쓸 수 있는지

## 판정 상태

각 기사/게시물에는 아래 상태를 둡니다.

- `직접 보도`: 사업이나 캠페인이 직접 등장
- `조직 언급`: 서울환경연합/서울환경운동연합만 언급
- `캠페인명 매칭`: 캠페인 고유명사가 등장
- `키워드 관련`: 관련 키워드는 있지만 직접 관련 여부 확인 필요
- `타지부/전국조직 혼재 가능성`: 지역 지부 또는 전국 조직과 섞일 가능성 있음
- `무관`: 집계 제외
- `검토 필요`: 사람이 판단해야 함

## 중복 기준

- 서로 다른 언론사의 유사 제목은 별도 보도로 봅니다.
- 같은 언론사 안에서 같은 제목, 같은 canonical URL, 원문/포털 대체 링크가 있으면 중복 의심으로 봅니다.
- 중복 의심은 자동 제외하지 않고 사람이 최종 확정합니다.

## Google Sheets 구조

Google Sheets는 여러 탭으로 나눕니다.

### `users`

| column | 설명 |
| --- | --- |
| email | 사용자 이메일 |
| role | `admin` 또는 `viewer` |
| name | 표시 이름 |
| active | 사용 여부 |

### `plans`

| column | 설명 |
| --- | --- |
| year | 사업계획서 연도 |
| title | 문서 제목 |
| source_url | Google Docs 링크 |
| imported_at | 업로드 시각 |
| raw_text_file | 원문 export 보관 위치 또는 참조 |

### `programs`

| column | 설명 |
| --- | --- |
| program_id | 사업 ID |
| year | 연도 |
| name | 사업명 |
| goal | 핵심목표 |
| change_goal | 변화목표 |
| indicators | 관찰지표/산출지표 요약 |
| owners | 내부 담당 |
| partners | 연대/협력 |
| active | 사용 여부 |

### `keywords`

| column | 설명 |
| --- | --- |
| keyword_id | 검색어 ID |
| keyword | 검색어 |
| type | 조직명, 캠페인명, 사업명, 협력기관명 |
| source | 사업계획서 추출 또는 관리자 입력 |
| active | 사용 여부 |
| notes | 잡음/주의사항 |

### `items`

| column | 설명 |
| --- | --- |
| item_id | 기사/게시물 ID |
| title | 제목 |
| url | 원문 URL |
| canonical_url | 중복 판정용 URL |
| source_name | 언론사/출처명 |
| source_type | 기사, 블로그, SNS, 기관, 단체 자체 |
| published_at | 게시일 |
| discovered_at | 수집일 |
| matched_keyword | 발견 검색어 |
| snippet | 검색 결과 스니펫 |
| ai_summary | AI 요약 |
| ai_basis | AI 매칭 근거 |
| review_status | 검토 상태 |
| include_in_press_count | 보도 횟수 포함 여부 |
| representative | 대표 기사 여부 |

### `matches`

| column | 설명 |
| --- | --- |
| match_id | 매칭 ID |
| item_id | 기사/게시물 ID |
| program_id | 사업 ID |
| match_type | 직접 관련, 간접 관련, 조직 언급, 검토 필요 |
| confidence_label | 높음, 중간, 낮음 |
| ai_reason | AI 판단 이유 |
| human_confirmed | 사람 확정 여부 |

### `fetch_runs`

| column | 설명 |
| --- | --- |
| run_id | 수집 실행 ID |
| started_at | 시작 시각 |
| finished_at | 종료 시각 |
| status | 성공, 일부 실패, 실패 |
| trigger | 자동, 수동 |
| queries_run | 실행한 검색어 수 |
| items_found | 발견 수 |
| items_created | 신규 저장 수 |
| items_updated | 갱신 수 |
| error_message | 오류 메시지 |

## 화면 구성

### 보기 전용 화면

- 전체 기사/게시물 목록
- 검색어, 출처 유형, 연도, 검토 상태 필터
- 조직 전체 언급 현황
- 캠페인/사업별 관련 기사
- 보도 횟수 집계
- 대표 기사 목록
- 평가용 요약 문장

### 관리자 화면

- 사업계획서 업로드
- 사업 목록 확인/수정
- 검색어 관리
- 수동 기사 수집 실행
- 수집 로그 확인
- 기사/게시물 검토
- AI 매칭 결과 확정
- 보도 횟수 포함 여부 변경
- 대표 기사 지정

## 수집 방식

하루 2회 자동 수집합니다.

- 오전 1회
- 저녁 1회
- 관리자 수동 재수집 가능

수집 경로:

- Google News RSS
- 네이버 뉴스 검색
- 네이버 웹 검색
- 언론사별 site 검색

이전 누락을 줄이기 위해 단일 검색 경로에 의존하지 않습니다.

## 우선 구현 순서

### 1단계: Sheets 기반 운영 MVP

- Google Sheets 탭 구조 만들기
- 사업계획서 텍스트 업로드/파싱
- 검색어 후보 추출
- 기사 수집 스크립트
- 기사 목록 저장
- 관리자 검토 화면
- 보기 전용 대시보드

### 2단계: AI 매칭

- 기사 요약
- 사업 후보 추천
- 직접/간접/조직 언급 판정
- 사람이 확정할 수 있는 근거 표시

### 3단계: 자동 운영

- Apps Script 하루 2회 트리거
- 수집 실패 로그
- 관리자 수동 재수집
- 보도 횟수/사회적 확산 요약

### 4단계: 확장 판단

Sheets가 느려지거나 관리가 어려워지면 DB 이전을 검토합니다.

## 성공 기준

- 관리자가 연간 사업계획서를 업로드하면 사업과 캠페인 후보가 추출됩니다.
- `서울환경연합`, `서울환경운동연합`, 주요 캠페인명을 기준으로 기사/게시물이 수집됩니다.
- 언론사 기사와 블로그/SNS/기관 게시물이 분리됩니다.
- 보도 횟수에는 언론사 기사만 들어갑니다.
- 같은 언론사 안의 중복 링크는 중복 의심으로 표시됩니다.
- 사람이 집계 포함 여부와 대표 기사 여부를 확정할 수 있습니다.
- 보기 전용 사용자는 결과를 볼 수 있지만 수정할 수 없습니다.
- 고정 비용 없이 또는 거의 없이 운영할 수 있습니다.

## 아직 열어둘 질문

- 오전/저녁 자동 수집 시각을 정확히 몇 시로 할지
- 관리자 로그인은 Google 계정 기반으로 할지, 간단한 공유 링크 + 이메일 목록으로 시작할지
- AI 요약을 어떤 API로 할지, 비용 상한을 어떻게 둘지
- Google Sheets 행 수가 어느 정도를 넘으면 DB 이전을 검토할지
