# 뉴스 대시보드 운영 기획서

## 한 줄 요약

서울환경연합의 연간 사업계획서를 기준으로, 조직과 캠페인이 언론사 기사에서 어떻게 언급되는지 모으고, 성과 평가에 쓸 수 있게 정리하는 저비용 뉴스 아카이브입니다.

## 왜 필요한가

현재는 구글 알리미와 잔디로 단체 관련 뉴스가 흘러오지만, 시간이 지나면 아카이빙되지 않아 평가 시점에 보도 횟수와 대표 근거를 다시 찾기 어렵습니다.

특히 사업계획서에는 여러 사업과 캠페인이 있고, 일부 사업은 언론보도 횟수와 별도 취재 여부를 함께 설명해야 합니다. 따라서 기사를 단순히 세는 도구가 아니라, 관련 기사와 품질 근거를 함께 남기는 운영 도구가 필요합니다.

## 기본 방향

우선순위는 `기사 중심 아카이브`입니다.

1. 언론사 기사 후보만 저장합니다.
2. 각 기사에 관련 조직, 관련 캠페인, 관련 사업 후보를 붙입니다.
3. 자동판정으로 관련 여부와 품질을 먼저 채우고, 관리자는 애매한 항목만 보정합니다.
4. 사업별 대시보드는 저장된 기사 위에서 분석 화면으로 제공합니다.

사업별 화면만 먼저 만드는 방식은 후순위입니다. 데이터가 먼저 잘 모여야 사업별 성과 화면도 신뢰할 수 있습니다.

## 운영 비용 기준

고정 비용을 최소화합니다.

- 화면: Vercel 무료 범위에서 시작
- 저장소: Google Sheets를 DB처럼 사용
- 자동 수집: Google Apps Script 시간 트리거
- 수집 주기: 하루 2회, 오전 1회와 저녁 1회
- 수동 수집: 관리자 화면에서 버튼으로 재수집 가능
- 향후 이전: Sheets가 느려지거나 데이터가 너무 많아지면 그때 Postgres/Supabase 같은 DB로 이전

처음부터 별도 DB를 붙이지 않습니다. 소규모 조직에서는 Google Sheets가 비개발자 관리와 비용 측면에서 더 적합합니다.

## 사용자와 권한

권한은 두 단계만 둡니다.

### admin

- 관리자 페이지 접근
- 사업계획서 업로드
- 검색어 추가, 수정, 비활성화
- 수동 기사 수집 실행
- 기사 검토 상태 변경
- 기사 품질 보정
- 대표 기사 지정
- AI 매칭 결과 확정 또는 수정

### viewer

- 대시보드 조회
- 기사 목록 조회
- 사업별 요약 조회
- 수정 불가

사업 담당자별 권한은 만들지 않습니다. 조직 규모상 `admin/viewer` 두 단계가 충분합니다.

## 사업계획서 관리

사업계획서는 매년 2월 전후 작성되고 이후 거의 업데이트되지 않습니다.

- 매년 관리자가 새 Google Docs 링크 또는 PDF/DOCX 파일을 업로드합니다.
- 자동 동기화는 하지 않습니다.
- 업로드한 내용은 해당 연도의 사업계획서 스냅샷으로 저장합니다.
- 연도별로 사업 목록, 활동명, 변화목표, 관찰지표, 산출지표, 담당자, 협력기관을 추출합니다.

## 수집 범위

이번 프로젝트에서는 언론사 기사만 저장합니다.

### 제외하는 것

- 서울환경연합 홈페이지
- 기관/공공기관 게시물
- SNS
- 블로그
- 단체 자체 게시물

### 집계 기준

- `보도 횟수`에는 검토 상태가 `관련`인 언론사 기사가 자동 포함됩니다.
- 별도 집계 승인 칸은 두지 않습니다.

## 기사 저장 범위

원문 전체는 저장하지 않습니다.

저장하는 정보:

- 제목
- URL
- 언론사 또는 출처명
- 게시일
- 검색어
- 스니펫
- 자동판정 근거
- 검토 상태
- 기사 품질

필요하면 원문에서 짧은 근거 문장 일부만 저장합니다. 평가나 최종 확인은 원문 링크를 열어 사람이 확인합니다.

## 검색어 생성 기준

사업계획서 전체 문장을 키워드로 쓰지 않습니다. 그러면 너무 많은 기사가 딸려옵니다.

### 항상 검색하는 조직명

- 서울환경연합
- 서울환경운동연합

### 자동 추출할 후보

사업계획서에서 고유명사 캠페인/브랜드명을 후보로 추출합니다.

예:

- 플라스틱방앗간
- 시티트리클럽
- 씨앗의숲

### 관리자가 할 수 있는 일

- 검색어 추가
- 검색어 수정
- 검색어 비활성화
- 검색어별 유형 지정: 조직명, 캠페인명

## 환경연합 유사 검색어 처리

`환경운동연합`, `환경연합`은 전국 조직과 지역 조직이 많아 잡음이 큽니다.

예:

- 파주환경운동연합
- 부산환경연합
- 지역 환경운동연합 지부

따라서 기본 원칙은 다음과 같습니다.

- `서울환경연합`, `서울환경운동연합`은 저장 후보로 봅니다.
- 지역 지부명만 있는 기사는 서울환경연합 성과로 자동 집계하지 않습니다.
- 다만 `시티트리클럽`, `플라스틱방앗간`, `씨앗의숲` 같은 관리 중인 캠페인 고유명사가 함께 나오면 저장합니다.
- 이 경우 `캠페인명 매칭`, `지역/전국 조직 혼재 가능성`으로 표시하고 사람이 검토합니다.

## AI 매칭 기준

자동판정 추천 + 관리자 보정입니다.

1. 규칙 기반 1차 후보 매칭
   - 조직명
   - 캠페인명
2. 자동 2차 판정
   - 어떤 사업과 관련 있는지
   - 관련인지 검토가 필요한지
   - 조직 전체 언급인지
   - 왜 그렇게 판단했는지

3. 관리자 보정
   - 관련 여부
   - 대표 기사 여부
   - 중복 여부
   - 기사 품질

## 판정 상태

각 기사에는 아래 검토 상태를 둡니다.

- `관련`: 언론보도 횟수에 자동 포함
- `검토 필요`: 조직명 또는 캠페인명은 잡혔지만 관리자가 확인해야 함
- `무관`: 저장되더라도 보도 횟수에서 제외

판정 근거에는 조직명 직접 언급, 조직명+캠페인명 직접 언급, 캠페인명 단독 언급, 다른 지역 환경연합 가능성 등을 남깁니다.

## 중복 기준

- 서로 다른 언론사의 유사 제목은 별도 보도로 봅니다.
- 같은 언론사 안에서 같은 제목, 같은 canonical URL, 원문/포털 대체 링크가 있으면 중복 의심으로 봅니다.
- 중복 의심은 자동 제외하지 않고 사람이 최종 확정합니다.

## Google Sheets 구조

Google Sheets는 여러 탭으로 나눕니다.

### `users`

| column | 설명 |
| --- | --- |
| email | 사용자 이메일 |
| role | `admin` 또는 `viewer` |
| name | 표시 이름 |
| active | 사용 여부 |

### `plans`

| column | 설명 |
| --- | --- |
| year | 사업계획서 연도 |
| title | 문서 제목 |
| source_url | Google Docs 링크 |
| imported_at | 업로드 시각 |
| raw_text_file | 원문 export 보관 위치 또는 참조 |

### `programs`

| column | 설명 |
| --- | --- |
| program_id | 사업 ID |
| year | 연도 |
| name | 사업명 |
| goal | 핵심목표 |
| change_goal | 변화목표 |
| indicators | 관찰지표/산출지표 요약 |
| owners | 내부 담당 |
| partners | 연대/협력 |
| active | 사용 여부 |
| updated_at | 관리자 수정 시각 |

### `keywords`

| column | 설명 |
| --- | --- |
| keyword_id | 검색어 ID |
| keyword | 검색어 |
| type | 조직명 또는 캠페인명 |
| source | 사업계획서 추출 또는 관리자 입력 |
| active | 사용 여부 |
| notes | 잡음/주의사항 |
| updated_at | 관리자 수정 시각 |

### `items`

| column | 설명 |
| --- | --- |
| item_id | 기사 ID |

| url | 원문 URL |
| canonical_url | 중복 판정용 URL |
| source_name | 언론사명 |
| source_type | media |
| published_at | 게시일 |
| discovered_at | 수집일 |
| matched_keyword | 발견 검색어 |
| snippet | 검색 결과 스니펫 |
| ai_summary | 현재 미사용 |

| representative | 대표 기사 여부 |
| quality | 기사 품질 |
| quality_basis | 품질 자동분류 근거 |

### `matches`

| column | 설명 |
| --- | --- |

| confidence | 점수 또는 신뢰도 |

| column | 설명 |
| --- | --- |

| finished_at | 종료 시각 |
| status | 성공, 일부 실패, 실패 |

| queries_run | 실행한 검색어 수 |
| items_found | 발견 수 |
| items_created | 신규 저장 수 |
| items_updated | 갱신 수 |

## 화면 구성

### 보기 전용 화면

- 전체 기사 목록
- 검색어, 언론사, 사업, 품질, 검토 상태 필터
- 조직 전체 언급 현황
- 캠페인/사업별 관련 기사
- 보도 횟수 집계
- 대표 기사 목록

### 관리자 화면

- 사업계획서 업로드
- 사업 목록 확인/수정
- 검색어 관리
- 수동 기사 수집 실행
- 수집 로그 확인
- 기사 검토
- 자동판정 결과 보정
- 대표 기사 지정

## 수집 방식

하루 2회 자동 수집합니다.

- 오전 1회
- 저녁 1회
- 관리자 수동 재수집 가능

수집 경로:

- Google News RSS
- 네이버 뉴스 검색
- Google News의 네이버/다음 뉴스 제한 검색

이전 누락을 줄이기 위해 단일 검색 경로에 의존하지 않습니다.

## 우선 구현 순서

### 1단계: Sheets 기반 운영 MVP

- Google Sheets 탭 구조 만들기
- 사업계획서 Google Docs 링크/PDF/DOCX 업로드와 파싱
- 검색어 후보 추출
- 기사 수집 스크립트
- 기사 목록 저장
- 관리자 검토 화면
- 보기 전용 대시보드

### 2단계: 자동판정

- 사업 후보 추천
- 관련/검토 필요 판정
- 품질 자동 추천
- 관리자가 보정할 수 있는 근거 표시

### 3단계: 자동 운영

- Apps Script 하루 2회 트리거
- 수집 실패 로그
- 관리자 수동 재수집
- 보도 횟수 요약

### 4단계: 확장 판단

Sheets가 느려지거나 관리가 어려워지면 DB 이전을 검토합니다.

## 성공 기준

- 관리자가 연간 사업계획서를 업로드하면 사업과 캠페인 후보가 추출됩니다.
- `서울환경연합`, `서울환경운동연합`, 주요 캠페인명을 기준으로 언론사 기사만 수집됩니다.
- 홈페이지, 기관, SNS, 블로그는 저장되지 않습니다.
- 보도 횟수에는 검토 상태가 `관련`인 언론사 기사가 자동으로 들어갑니다.
- 같은 언론사 안의 중복 링크는 중복 의심으로 표시됩니다.
- 관리자가 관련성, 품질, 대표 기사 여부를 보정할 수 있습니다.
- 보기 전용 사용자는 결과를 볼 수 있지만 수정할 수 없습니다.
- 고정 비용 없이 또는 거의 없이 운영할 수 있습니다.

## 아직 열어둘 질문

- 오전/저녁 자동 수집 시각을 정확히 몇 시로 할지
- 품질 자동분류를 규칙 기반으로 유지할지, 별도 AI API를 붙일지
- Google Sheets 행 수가 어느 정도를 넘으면 DB 이전을 검토할지
