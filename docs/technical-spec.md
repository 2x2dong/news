# 뉴스 대시보드 명세

## 목표

서울환경연합과 서울환경운동연합, 그리고 사업계획서에서 추출한 고유 캠페인명이 언론과 외부 게시물에서 언급되는 흐름을 저장하고 평가 근거로 정리한다.

## 기본 구조

- 화면: Vercel 정적 배포
- 저장소: Google Sheets
- 쓰기 연결: Google Apps Script 웹앱
- 정기 실행: Google Apps Script 시간 트리거, 오전 1회와 저녁 1회
- 운영 권한: 관리자와 보기 전용 두 단계

## 사용자 권한

| 권한 | 가능 작업 |
| --- | --- |
| admin | 사업계획서 업로드, 검색어 관리, 기사 검토, 집계 포함 여부 변경, 시트 동기화 |
| viewer | 대시보드와 기사 목록 조회 |

## 데이터 원칙

- 언론사 기사는 `source_type=media`로 저장한다.
- 보도 횟수에는 `source_type=media`, `review_status=related`, `include_in_press_count=true`인 항목만 포함한다.
- 블로그, SNS, 기관 게시물, 단체 자체 게시물은 저장하되 보도 횟수에는 넣지 않는다.
- 원문 전체를 저장하지 않고 제목, URL, 게시일, 스니펫, 요약, 판정 근거만 저장한다.
- 서로 다른 언론사의 유사 제목은 중복으로 보지 않는다.
- 같은 언론사 안에서 같은 제목이나 같은 canonical URL이 있을 때만 중복 의심으로 표시한다.

## 검색어 원칙

- 기본 조직명: 서울환경연합, 서울환경운동연합
- 사업계획서에서 고유명사 캠페인명을 추출한다.
- 예시: 플라스틱방앗간, 시티트리클럽, 씨앗의숲
- 일반적인 `환경연합`, `환경운동연합`만 있는 지역 지부 기사는 자동 집계하지 않는다.
- 지역 지부명과 캠페인명이 함께 있는 경우 저장하고 사람이 검토한다.

## Google Sheets 탭

| 탭 | 용도 |
| --- | --- |
| users | 사용자 권한 |
| plans | 연도별 사업계획서 스냅샷 |
| programs | 사업 단위 |
| keywords | 검색어와 캠페인명 |
| items | 기사와 외부 게시물 |
| matches | 기사와 사업의 연결 |
| fetch_runs | 수집 실행 기록 |

## 구현 단계

1. 정적 대시보드와 로컬 저장 기반 관리자 화면
2. Google Sheets 탭 생성과 Apps Script 연결
3. 오전/저녁 자동 수집 기록
4. 기사 후보 수집기 확장
5. AI 요약과 사업 매칭 자동 추천
