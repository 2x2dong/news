# 뉴스 대시보드 내부 사용 가이드

마지막 업데이트: 2026-07-01

이 문서는 서울환경연합 내부 활동가가 뉴스 대시보드를 함께 운영하기 위한 사용 안내입니다. 코딩 지식이 없어도 매일 기사 확인, 사업 분류, 품질 판단, 보고용 공유까지 할 수 있도록 정리했습니다.

## 1. 이 대시보드의 목적

뉴스 대시보드는 서울환경연합과 캠페인명이 언론 기사에 어떻게 등장했는지 모아두는 아카이브입니다. 평가 시기에 “언론보도 몇 건이 있었는지”, “어떤 사업이 사회적으로 확산되었는지”, “대표로 보여줄 만한 기사가 무엇인지”를 바로 확인하기 위해 씁니다.

이 대시보드는 공개 언론사 기사만 다룹니다. 서울환경연합 홈페이지, 기관 홈페이지, SNS, 블로그 글은 이번 사업의 집계 대상이 아닙니다.

## 2. 접속과 권한

운영 주소: https://seoulkfem-news.vercel.app

권한은 두 가지입니다.

| 구분 | 쓰는 사람 | 할 수 있는 일 |
| --- | --- | --- |
| 관리자 | 사무처 내부 활동가 | 기사 검토, 사업/품질 수정, 검색어 관리, 사업계획서 업로드, 기사 수집 |
| 보기 전용 | 이사회, 회원, 외부 공유 대상 | 기사 목록과 지표 열람 |

관리자 비밀번호는 별도 내부 채널로 공유합니다. 이 문서나 외부에 공개되는 저장소에는 비밀번호를 적지 않습니다.

## 3. 매일 확인할 일

하루에 한 번 정도 아래 순서로 보면 됩니다.

1. 대시보드에 접속합니다.
2. 관리자 비밀번호로 로그인합니다.
3. 상단의 `확인 필요` 숫자를 봅니다.
4. 노란색으로 강조된 기사부터 확인합니다.
5. 기사 원문을 열어 관련 여부, 사업명, 기사 품질을 정리합니다.
6. 중요하다고 판단되는 기사는 `대표 기사`로 표시합니다.

자동 수집은 매일 오전 9시대와 오후 6시대에 한 번씩 돌아가도록 설정되어 있습니다. 급하게 확인해야 할 때는 관리자 화면의 `기사 수동 수집` 버튼을 누르면 됩니다. 수집 중에는 진행 상태창이 뜨므로 여러 번 누르지 않아도 됩니다.

## 4. 상단 지표 읽는 법

| 지표 | 뜻 |
| --- | --- |
| 관련 기사 | 검토 상태가 `관련`인 언론사 기사 수입니다. 보도 횟수 집계 기준입니다. |
| 대표 기사 | 보고서나 평가 문장에 대표로 쓸 만하다고 표시한 기사입니다. |
| 확인 필요 | 검토 상태, 사업 분류, 기사 품질 중 사람이 확인해야 할 항목이 남아 있는 기사입니다. 노란색 강조 기사 수와 같은 기준입니다. |
| 중복 의심 | 같은 언론사 안에서 같은 기사로 보이는 항목입니다. 다른 언론사의 비슷한 기사는 중복이 아닙니다. |

`확인 필요` 숫자가 0에 가까워질수록 검토가 잘 끝난 상태입니다.

## 5. 기사 검토 기준

기사 목록에서 원문을 열고 아래 기준으로 판단합니다.

### 관련 여부

| 상태 | 기준 |
| --- | --- |
| 관련 | 서울환경연합, 서울환경운동연합, 또는 우리 캠페인명이 기사 안에서 실제로 다뤄진 경우 |
| 검토 필요 | 키워드는 잡혔지만 서울환경연합 관련 기사인지 확실하지 않은 경우 |
| 무관 | 다른 지역 환경연합, 다른 단체, 단순 동명이거나 사업과 관련 없는 경우 |

다른 언론사가 같은 보도자료를 비슷하게 쓴 경우도 각각 별도 언론보도로 봅니다. 단, 같은 언론사 안에서 같은 기사가 여러 링크로 잡힌 경우만 중복으로 봅니다.

### 사업 분류

기사의 사업명은 가능한 한 실제 사업 목록에 맞춥니다. 사업계획서에 없는 갑작스러운 현장 대응 기사라면 아래 순서로 처리합니다.

1. 내용이 특정 팀의 정책 대응과 맞으면 해당 팀의 정책 대응 사업으로 분류합니다.
2. 어느 팀인지 애매하거나 사업계획서에 없는 이슈면 `기타 정책 대응`으로 둡니다.
3. 이후 같은 이슈가 반복되면 관리자 화면에서 사업을 수동 추가할 수 있습니다.

팀명은 사업 목록의 분류와 같습니다.

- 생태도시
- 기후행동
- 자원순환
- 시민참여
- 모금
- 기타

### 기사 품질

품질은 기사 숫자와 별개로, 보도 영향력을 설명하기 위한 보조 기준입니다.

| 품질 | 기준 |
| --- | --- |
| 상 | 별도 취재, 분석, 단독, 르포, 현장 취재처럼 언론사가 독자적으로 다룬 기사 |
| 중 | 보도자료를 바탕으로 하되 활동가 인터뷰, 추가 설명, 맥락 설명이 붙은 기사 |
| 하 | 보도자료를 거의 그대로 옮긴 기사, 행사 안내성 기사, 사진과 캡션 중심 기사 |
| 미분류 | 아직 사람이 품질을 확인하지 않은 기사 |

품질이 애매하면 원문을 열고 기사 안에 기자의 별도 취재나 추가 질문이 있는지 봅니다.

## 6. 필터 사용법

왼쪽 필터에서 필요한 기사만 좁혀볼 수 있습니다.

| 필터 | 언제 쓰나 |
| --- | --- |
| 검색 | 제목, 언론사, 키워드, 사업명으로 찾을 때 |
| 검토 상태 | 관련, 검토, 대표, 중복만 따로 볼 때 |
| 출처 | 특정 언론사 기사만 볼 때 |
| 주요 언론사만 보기 | 보고용으로 주요 언론 보도만 빠르게 확인할 때 |
| 팀명 | 생태도시, 기후행동, 자원순환 등 팀별 기사만 볼 때 |
| 사업명 | 특정 사업 기사만 볼 때 |
| 기사 품질 | 상/중/하/미분류 기사만 볼 때 |

팀명 필터는 사업 목록의 `분류` 기준과 같습니다. 사업명 필터보다 넓게 보고 싶을 때 팀명 필터를 먼저 쓰면 좋습니다.

## 7. 검색어 관리

기본 검색어는 `서울환경연합`, `서울환경운동연합`입니다. 사업계획서를 업로드하면 `시티트리클럽`, `플라스틱방앗간`, `씨앗의숲`처럼 고유한 캠페인명이 검색어로 추천됩니다.

관리자는 검색어를 추가하거나 비활성화할 수 있습니다.

- 너무 넓어서 무관 기사가 많이 잡히면 비활성화합니다.
- 캠페인명처럼 고유하고 중요한 단어는 활성화합니다.
- `환경운동연합`처럼 전국 조직 전체를 가리키는 검색어는 서울 기사만 잡기 어려우므로 주의합니다.

검색어를 비활성화해도 기존 기사가 삭제되지는 않습니다. 앞으로의 수집에서 그 검색어를 덜 쓰게 되는 것입니다.

## 8. 사업 목록 관리

사업 목록은 사업계획서 업로드로 자동 생성되지만, 운영 중 새 이슈가 생기면 수동으로 추가할 수 있습니다.

사업을 추가할 때는 아래를 입력합니다.

| 항목 | 입력 기준 |
| --- | --- |
| 사업명 | 활동가가 실제로 알아볼 수 있는 이름 |
| 분류 | 생태도시, 기후행동, 자원순환, 시민참여, 모금, 기타 중 선택 |
| 설명 | 분류 기준이나 메모 |

더 이상 쓰지 않는 사업은 비활성화할 수 있습니다. 비활성화해도 기존 기사 기록은 사라지지 않습니다.

## 9. 사업계획서 업로드

사업계획서는 매년 2월 전후 작성 후 크게 바뀌지 않는 전제로 운영합니다. 새해 사업계획서가 나오면 관리자 화면에서 업로드합니다.

가능한 형식은 다음과 같습니다.

- Google Docs 링크
- PDF 파일
- DOCX 파일

업로드하면 대시보드가 사업명과 캠페인명 후보를 자동 추출합니다. 자동 추출 결과는 완벽하지 않을 수 있으므로, 관리자가 사업 목록과 검색어 목록을 확인해 활성/비활성 상태를 정리합니다.

## 10. 기사 수동 추가

자동 수집에 없는 기사를 발견했을 때는 관리자 화면에서 직접 추가할 수 있습니다. 가능한 한 제목, 언론사, 날짜, URL, 요약을 채워 넣습니다.

직접 추가한 기사는 처음에는 확인이 필요한 항목으로 보는 것이 좋습니다. 원문을 열어 관련 여부, 사업명, 품질을 다시 확정합니다.

## 11. 보기 전용 링크 공유

이사회 보고나 회원 공유에는 관리자 화면이 아니라 `보기 링크`를 복사해 공유합니다.

보기 전용 링크를 받은 사람은 기사와 지표를 볼 수 있지만 수정은 할 수 없습니다. 외부 공유 전에는 `확인 필요` 기사와 `미분류` 품질이 너무 많이 남아 있지 않은지 먼저 확인하는 것이 좋습니다.

## 12. 데이터가 사라지는 것처럼 보일 때

최근 버전부터 관리자 확인값은 자동 수집이나 코드 업데이트로 덮어쓰지 않도록 보존됩니다. 그래도 화면이 이상하면 아래 순서로 확인합니다.

1. 새로고침합니다.
2. 관리자 로그인 상태인지 확인합니다.
3. 관리자 화면에서 `구글시트 불러오기`를 누릅니다.
4. 방금 수정한 값이 반영되지 않았다면 `Google Sheets 저장`을 누릅니다.
5. 그래도 이상하면 최근 수정한 기사 제목, 바뀐 값, 발생 시간을 기록해 담당자에게 공유합니다.

Google Sheets를 직접 수정할 수도 있지만, 가능하면 대시보드 화면에서 수정하는 것을 권장합니다. 화면에서 수정하면 필요한 보존 표시가 함께 저장됩니다.

## 13. 운영 원칙

- 보도 횟수는 `관련`으로 확인된 언론사 기사만 집계합니다.
- 노란색 기사는 먼저 확인합니다.
- 품질은 기사 수를 줄이기 위한 기준이 아니라, 영향력을 설명하기 위한 보조 기준입니다.
- 사업계획서에 없는 이슈도 버리지 말고 정책 대응 또는 기타로 분류합니다.
- 관리자 비밀번호는 내부에서만 공유합니다.
- 보기 전용 링크는 이사회나 회원 공유에 사용합니다.

## 14. 빠른 체크리스트

정기 점검 때 아래만 확인해도 됩니다.

- `확인 필요` 숫자가 너무 많이 쌓이지 않았는가?
- 새로 들어온 기사 중 무관 기사를 `무관`으로 바꿨는가?
- 관련 기사의 사업명과 팀명이 맞는가?
- 미분류 품질을 상/중/하로 정리했는가?
- 대표 기사를 표시했는가?
- 너무 넓거나 잡음이 많은 검색어를 비활성화했는가?
- 공유 전 보기 전용 링크로 화면을 확인했는가?
