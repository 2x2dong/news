# 시티트리클럽 기사 수집 기준

## 이전 누락 원인

초기 대시보드는 실제 수집기가 아니라 MVP 샘플 데이터였습니다. 그래서 아래 문제가 있었습니다.

- 검색 쿼리가 너무 좁았습니다. `시티트리클럽`, `가로수 기록 커뮤니티맵`, `서울환경연합 시티트리클럽` 정도만 보면서 언론사별 site 검색을 하지 않았습니다.
- Google News RSS만 보면 빠지는 언론사가 있습니다. 특히 Btv뉴스와 환경과조경은 네이버 검색 결과에서는 보이지만 Google News RSS에서는 안정적으로 잡히지 않았습니다.
- Google News RSS 결과는 원문 URL 대신 `news.google.com` 링크를 주는 경우가 있어, 원문 도메인만 보고 필터링하면 소셜임팩트뉴스 같은 결과가 빠질 수 있습니다.
- 같은 언론사의 대체 게재 링크와 원문 링크를 구분하지 않았습니다. 경향신문 네이트 링크와 경향신문 직접 링크는 같은 언론사 안의 중복 의심으로 봐야 합니다.
- 관련성 숫자 점수는 실제 계산 모델이 아니라 임의 샘플값이어서 판단 기준으로 부적절했습니다.

## 수정한 수집 전략

`scripts/collect-citytreeclub-news.mjs`는 아래 경로를 함께 확인합니다.

- Google News RSS
- 네이버 뉴스 검색
- 네이버 웹 검색
- 정확 문구 검색: `시티트리클럽`, `가로수 기록 커뮤니티맵`
- 맥락어 검색: `서울환경연합 시티트리클럽`, `나무의 권리 시티트리클럽`
- 언론사별 site 검색: `news.skbroadband.com`, `lak.co.kr`, `ekoreanews.co.kr`

## 집계 기준

- 2026년 1월 1일 이후 공개 언론사 기사만 보도 후보로 봅니다.
- 서울환경연합 홈페이지, 서울시공익활동지원센터, 블로그, 인스타그램, X 등은 보도 횟수에서 제외합니다.
- 서로 다른 언론사의 유사 제목은 별도 보도로 둡니다.
- 같은 언론사 안의 같은 제목 또는 같은 canonical URL만 중복 의심으로 봅니다.
- `직접 보도`는 제목/내용에 시티트리클럽 또는 가로수 기록 커뮤니티맵이 직접 나타나는 경우입니다.
- `키워드 관련`은 가로수, 나무의 권리, 서울환경연합 등 주변 키워드는 맞지만 시티트리클럽 직접 보도인지 확인이 필요한 경우입니다.
- `대체 게재 링크`는 같은 언론사의 원문/포털 게재 링크처럼 같은 보도로 볼 가능성이 큰 경우입니다.

## 실행 방법

```bash
node scripts/collect-citytreeclub-news.mjs
```

생성 파일:

- `data/discovery-report.json`
- `data/discovery-report.md`

수집기는 네이버가 일부 요청을 403으로 막아도 멈추지 않고 나머지 경로로 리포트를 생성합니다. 최종 집계 전에는 사람이 원문 링크와 언론사 여부를 확인해야 합니다.
