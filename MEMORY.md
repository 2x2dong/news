# 만들기 기록 - 뉴스 대시보드

> 이 파일은 이 MVP를 어떤 과정으로 만들었는지 남기는 기록입니다.
> 나중에 복기하거나 사회혁신 AI 에이전트 워크플로 사례로 정리할 때 씁니다.

## 한눈에 보기
- 무엇을: 애드보커시 단체는 사업 성과지표로 언론보도 횟수를 정하는 경우가 많지만, 현재 단체 관련 키워드를 구글 알리미로 받아 업무 메신저 잔디에 흘려보내는 방식이라 보도 내역이 아카이빙되지 못하고 휘발된다. 평가 시기에는 실제 언론보도 횟수와 근거 기사를 측정하기 어렵다.
- 누구를 위해: 사업 담당자인 활동가, 평가해야 하는 동료 및 사무처장
- 핵심 흐름: 활동명/키워드 입력 -> 공개 기사 목록 정리 -> AI 관련성/요약 초안 -> 담당자 검토 -> 평가 근거로 활용
- 스택/도구: Codex, 공개 웹 검색 또는 수동 입력 기사 목록, 로컬 대시보드
- 시작: 2026-06-27

## 기록

### 2026-06-27 문제 정의
- **정한 것 / 한 것**: 뉴스 대시보드의 문제정의와 3시간 MVP 범위를 확정하고 PLAN.md 및 workshop.json 초안을 생성했다.
- **왜**: 언론보도 알림이 휘발되어 사업 평가 시 근거가 부족해지는 문제를 공개 기사 기반 대시보드와 평가용 요약 리포트로 줄이기 위해서다.
- **어떻게**: 사회혁신가, 개발자, 프로젝트명, 업무 문제, 자료 상태, 결과물 형태, 키워드, 사람 검토 지점, 성공/실패 기준을 표준 필드로 정규화했다.
- **막힌 점 / 바꾼 점**: 공유된 Google 문서 본문은 직접 읽을 수 없어, 사용자가 제공한 대표 활동명과 핵심 키워드를 기준으로 MVP 범위를 정했다.
- **배운 것 / 다음**: 구현자는 PLAN.md 기준으로 대시보드 MVP를 만들고, 구현 중 결정과 검증 결과를 MEMORY.md에 append-only로 남긴다.

### 2026-06-27 구현
- **정한 것 / 한 것**: `index.html`, `styles.css`, `app.js`로 브라우저에서 바로 열 수 있는 정적 뉴스 성과 대시보드 MVP를 만들었다.
- **왜**: 3시간 MVP에서는 자동 크롤링, 잔디 연동, 운영 DB보다 공개 기사 묶음을 검토 가능한 한 화면으로 정리하는 것이 가장 빠르게 현장 효용을 보여주기 때문이다.
- **어떻게**: 시티트리클럽 관련 공개 기사 샘플을 내장하고, 기사별 관련성, 대표 기사, 집계 포함 여부, 중복 의심, 키워드 근거, 평가용 요약 문장, 수동 기사 추가, JSON 내보내기를 구현했다.
- **막힌 점 / 바꾼 점**: 기사 자동 수집은 제외 범위로 두고, 공개 URL과 수동 입력 샘플을 기준으로 검토 가능한 대시보드에 집중했다.
- **배운 것 / 다음**: 실제 운영으로 확장하려면 구글 알리미/잔디 링크 자동 추출, 월별 아카이브, 중복 병합 기준, 사업별 비교 화면을 다음 단계로 붙이면 된다.

### 2026-06-27 구현 보정
- **정한 것 / 한 것**: 2026년 1월 1일 이후 공개 언론사 기사 기준임을 화면에 명시하고, 서울환경연합 홈페이지와 서울시공익활동지원센터 같은 비언론사 출처 및 삭제 링크로 연결되는 경향신문 샘플을 제거했다.
- **왜**: 사업 성과지표의 언론보도 횟수는 언론사 기사만 집계해야 하며, 삭제 링크나 조직 자체 게시물은 평가 근거로 쓰면 안 되기 때문이다.
- **어떻게**: 샘플 데이터 버전을 올리고, 언론사 4곳의 공개 기사만 남겼으며, 중복 의심 로직을 같은 언론사 안에서 같은 기사로 보이는 경우에만 작동하도록 바꿨다.
- **막힌 점 / 바꾼 점**: 서로 다른 언론사가 같은 제목이나 내용을 낸 경우는 중복이 아니라 별도 보도로 집계하도록 바꿨다.
- **배운 것 / 다음**: 실제 검색 자동화를 붙일 때도 출처 유형 필터와 같은 언론사 내 중복 판정 기준을 먼저 적용해야 한다.

### 2026-06-27 수기 시트 대조
- **정한 것 / 한 것**: 사용자가 제공한 Google Sheets 수기 목록에서 2026년 1월 1일 이후 `분류=기사` 7건을 읽어 현재 대시보드와 비교하고, 빠져 있던 경향신문, Btv뉴스, 환경과조경, 이코리아 링크를 추가했다.
- **왜**: 사용자가 며칠 전에 직접 찾은 목록이 현장 기준 데이터이므로, MVP가 해당 언론사와 링크를 모두 찾을 수 있어야 하기 때문이다.
- **어떻게**: 수기 목록의 기사 7건을 앱 기본 데이터로 반영하고, URL 일부와 수기목록 행번호로도 검색되도록 검색 대상을 확장했다.
- **막힌 점 / 바꾼 점**: 경향신문 네이트 링크와 경향신문 직접 링크는 같은 언론사 안의 중복 의심으로 묶고, 보도 횟수 집계는 직접 링크 1건만 포함했다. 환경과조경 링크는 자동 요청에서 403을 반환하지만 수기 시트의 공개 링크로 보존했다.
- **배운 것 / 다음**: 자동 검색을 붙일 때도 수기 시트와 대조해 누락 링크, 대체 게재 링크, 같은 언론사 중복 링크를 분리해야 한다.

### 2026-06-27 수집 방식 보강
- **정한 것 / 한 것**: 숫자 관련성 점수를 화면에서 제거하고, `직접 보도`, `대체 게재 링크`, `키워드 관련` 같은 판정 근거로 바꿨다. 또한 `scripts/collect-citytreeclub-news.mjs`와 `docs/article-discovery.md`를 추가했다.
- **왜**: 기존 숫자 점수는 실제 모델 계산값이 아니라 임의 샘플값이어서 오해를 만들었고, 수기 시트는 지속 비교용이 아니라 누락 원인을 찾아 수집 방식을 고치기 위한 검증 자료였기 때문이다.
- **어떻게**: Google News RSS, 네이버 뉴스/웹 검색, 정확 문구 검색, 맥락어 검색, 언론사별 site 검색을 함께 돌리는 수집 리포트 생성 스크립트를 만들었다.
- **막힌 점 / 바꾼 점**: Google News RSS만 보면 Btv뉴스와 환경과조경이 빠지고, 원문 URL이 `news.google.com`으로 남아 소셜임팩트뉴스가 필터링될 수 있어 소스명 기반 언론사 판정도 추가했다.
- **배운 것 / 다음**: 수집 자동화는 단일 검색 API가 아니라 복수 검색 경로, 출처 유형 필터, 같은 언론사 내 중복 판정, 사람 검토 상태를 함께 가져야 한다.

### 2026-06-27 운영 기획 문서화
- **정한 것 / 한 것**: 지금까지 인터뷰에서 확정한 운영 방향을 `docs/vercel-google-sheets-product-plan.md`에 기획문서로 저장했다.
- **왜**: 사용자는 코딩 지식보다 운영 비용, 관리 편의성, 실제 단체 업무 흐름을 기준으로 의사결정해야 하므로, 개발 용어보다 운영 구조와 화면/시트 구성을 먼저 고정해야 한다.
- **어떻게**: 기사 중심 아카이브, 사업별 분석 화면, Vercel 화면, Google Sheets 저장소, admin/viewer 권한, 연 1회 사업계획서 수동 업로드, 하루 2회 자동 수집, AI 추천+사람 확정 흐름을 문서화했다.
- **막힌 점 / 바꾼 점**: Postgres DB 대신 Google Sheets를 먼저 쓰는 저비용 운영안으로 방향을 바꿨다.
- **배운 것 / 다음**: 다음 구현은 Google Sheets 탭 구조와 관리자/보기 전용 화면을 이 기획문서 기준으로 설계하면 된다.
